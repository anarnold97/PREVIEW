# **OpenShift CQA & Readability Assessment Tool**

The **CQA & Readability Assessment Tool** (`cqa_report.py`) is a local Python utility designed for OpenShift technical writers.

Instead of just checking the files you have recently modified, this tool runs a **comprehensive audit on an entire product directory or subdirectory**. It validates your AsciiDoc assemblies and their included modules against the CQA 2.1 guidelines, Red Hat/IBM style guides, and industry-standard readability metrics.

## **Features**

* **Smart Module Resolution:** It scans your assemblies, finds every `include::[]` statement, and automatically pulls in the active files from the `/modules` directory. It skips orphaned or unused modules.  
* **Interactive Selection:** Easily choose whether to audit the Virt or OADP documentation sets.  
* **Targeted Scanning:** Narrow down your audit to specific subdirectories (like `install` or `updating`) to assess just the book you are working on, saving time.  
* **CQA 2.1 Alignment:** Uses the [`vale-cqa.ini`](https://redhat.atlassian.net/wiki/spaces/cnv/pages/268599849/.vale-cqa.ini) configuration to enforce automated rules.  
* **Pinpoint Accuracy:** The terminal output shows you the exact word, phrase, or symbol in your file that triggered a rule violation.  
* **Readability Scoring:** Uses the Flesch-Kincaid grade level formula to assess content complexity. It intelligently strips out AsciiDoc code blocks, URLs, and attributes so they don't artificially inflate the reading age.

---

## **Prerequisites**

Before running the script, ensure you have the following installed on your local machine:

1. **Python 3.x:** To run the script.  
2. **Textstat (Python Library):** Used for calculating the reading age. Install it via pip:  

```bash
pip install textstat
```

3. **Vale CLI:** The linting engine. [See Vale Installation Instructions](https://www.google.com/search?q=https://vale.sh/docs/vale-cli/installation/).  
4. [**`vale-cqa.ini`:**](https://redhat.atlassian.net/wiki/spaces/cnv/pages/268599849/.vale-cqa.ini) The official CQA configuration file must be present in the root of the repository.

---

## **Usage**

You must run this script from the **root of the `openshift-docs` repository** so it can locate your configuration files and the correct directory structures.

1. Open your terminal and navigate to the root of the repo.  
2. Run the script:  

```bash
python3 cqa_report.py
```

3. The script will prompt you to select a documentation set:  

```
=== OpenShift CQA Directory Assessment ===
Which documentation set would you like to assess?
  1. Virt (/virt)
  2. OADP (/backup_and_restore/application_backup_and_restore)

Enter 1 or 2: 
```

4. After making your selection, the script will ask if you want to scan a specific subdirectory:  


```
Enter a specific subdirectory to scan within Virt.
For example, type 'install' to scan virt/install
Or just press Enter to scan ALL of Virt.

Subdirectory path: 
```

5. Enter your targeted directory (or press **Return** to scan the entire doc set). The script will map the files, run Vale, calculate readability, and output the report.

## **Understanding the Output**

The generated report is color-coded and split into three sections:

### **1. REQUIRED / NON-NEGOTIABLE (Migration Blockers)**

* **What it is:** Critical structural errors flagged by the `AsciiDocDITA` Vale rules. These represent AsciiDoc markup that has no direct equivalent in DITA 1.3.  
* **Action Required:** You **must** fix these issues before your content can be migrated to AEM.  
* **Example:** \> `modules/virt-example.adoc:14` \[AsciiDocDITA.DocumentTitle\] Document title is missing \-\> **`=`**

### **2. IMPORTANT / NEGOTIABLE (Style & Quality)**

* **What it is:** Stylistic warnings generated by the `RedHat` and `IBM` Vale packages (e.g., passive voice, non-standard capitalization, prohibited words).  
* **Action Required:** You **should** review and fix these to improve content quality, but they will not technically block migration.  
* **Example:** \> `virt/install/virt-installing.adoc:22` \[RedHat.PassiveVoice\] Avoid passive voice \-\> **`was installed`**

### **3. READABILITY & COMPLEXITY**

* **What it is:** An assessment using the Flesch-Kincaid Grade Level formula. The standard goal for technical writing is an **8th to 10th-grade reading level**.  
* **Action Required:** The script provides the average grade across your entire directory. If any individual file scores above a **Grade 12** (requiring a college-level reading comprehension), it will be flagged in yellow/red. Consider applying minimalism principles or breaking up long, complex sentences in those specific files.

## **Troubleshooting**

* **"Error: The 'textstat' library is missing."**

    You need to install the readability library. Run `pip install textstat`. If you are using a virtual environment, ensure you activate it before running the script.

* **"Error: 'vale-cqa.ini' not found in the current directory."**

    You are likely running the script from inside a subdirectory like `/virt`. Always run `cqa_report.py` from the root of the repository.

* **"Warning: Could not read virt/example.adoc to parse includes"**

    The script encountered a file permission error or an encoding issue when trying to read the assembly to find its modules. Ensure you have read access to the files.
